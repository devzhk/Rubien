# rubien-chat-renderer

The chat-transcript renderer for the Assistant sidebar: a self-contained HTML/JS bundle that renders streamed markdown + LaTeX **safely** inside a macOS `WKWebView`. Untrusted model/document text is treated as hostile — `marked` runs with raw HTML disabled, every rendered string passes through DOMPurify (pinned) before it touches `innerHTML`, a strict CSP `<meta>` makes remote loads impossible, and only `http:`/`https:` links are live. KaTeX is the vendored copy from `Sources/Rubien/Resources/` (fonts inlined as `data:` URIs), so math is byte-identical to the web reader and works fully offline. The Swift side talks to the page through the `window.RubienChat` API and the `chatReady` / `openExternalLink` / `copyCode` message handlers.

Math reaches KaTeX intact because `src/render.js` tokenizes TeX ahead of marked's own rules — `\(…\)` / `\[…\]` / `$…$` / `$$…$$`, display environments, and ` ```math ` / ` ```latex ` fences — re-emitting each formula as escaped text normalized to `\( \)` / `\[ \]` for the post-sanitize `renderMathInElement` pass. Without that, CommonMark strips the backslash delimiters (`\[` → `[`, so inline math shows as literal `(θ)`) and bare `=` / `-` lines inside a display formula become setext headings.

`npm install && npm run build` regenerates the committed `../../Sources/Rubien/Resources/ChatTranscript.html` (esbuild bundles `src/chat.js` + `src/render.js` + `src/chat.css`, then inlines the vendored KaTeX CSS/JS). `npm test` runs the `node --test` security suite (`test/security.test.js`, jsdom-backed) that asserts hostile payloads — `<script>`, `onerror` handlers, `javascript:`/`file:`/`data:` links, raw HTML, prompt-injection strings — all render inert. The `ChatTranscript.html` artifact is committed to the repo (same discipline as `scripts/note-editor/` → `NoteEditor.html`); rebuild and commit it whenever the renderer changes.
